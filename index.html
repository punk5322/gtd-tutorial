GTD System Tutorial

Interactive guide for Arsh's David Allen GTD setup

🎯 What is GTD?

Getting Things Done (GTD) is a productivity methodology created by David Allen. It helps you capture every commitment, organize it by context, and review it weekly.

The 5 Key Lists:

  • Inbox — Capture zone for everything
  • Next Actions — Tasks by context (@calls, @computer, @outside)
  • Projects — Multi-step outcomes with next actions
  • Someday/Maybe — Low priority ideas
  • Waiting For — Delegated items
1

📝 Capture Tasks

When a task or idea comes to mind, capture it immediately. Don't try to organize while capturing — just get it out of your head.

You say: "capture Finish the Q2 investor deck"
→ Sam adds to Inbox.md
Sam replies: "Captured: Finish the Q2 investor deck"
2

🔄 Process Inbox

At least once a day, go through your Inbox. For each item, decide:

  • Schedule — Add to Next Actions with a date
  • Someday — Move to Someday/Maybe
  • Trash — Delete it
  • Now — Do it now
You say: "process inbox"
→ Sam shows items, you choose what to do with each
3

📅 Today's Tasks

Ask for today's view to see what's due now.

You say: "what's due today"
Sam replies: "3 tasks today:
1. [Next Actions] Close Lucille AB deal
2. [Next Actions] Close STRM deal
3. [Projects] Create Series A deck"
4

⏳ Waiting For

Track things you've delegated to others.

You say: "waiting for Geoff re: STRM MSA"
→ Added to @waiting-for in Next Actions.md
Sam replies: "Logged: Geoff Cubitt — STRM MSA (sent 2026-04-16)"
5

📁 Projects

Multi-step outcomes need their own file with a next action.

You say: "new project Build mobile app"
→ Sam creates Projects/Mobile App.md
Sam asks: "What's the outcome? What's the first next action?"
6

📊 Weekly Review

Every Friday, Sam prompts you for a weekly review. Interactive walkthrough:

  1. Review Inbox items
  2. Check projects without next actions
  3. Review Someday/Maybe for激活items
  4. Set top 3 for next week
